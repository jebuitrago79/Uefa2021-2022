Diagrama de Flujo del Sistema

Este diagrama ilustra el flujo de ejecución del sistema desde que el usuario accede hasta que se comunican los endpoints, rutas y modelos de datos definidos con FastAPI, SQLModel y PostgreSQL/Supabase.

Mapa de Endpoints

El enrutamiento en este sistema comienza desde main.py, que delega el manejo de solicitudes a través de dos routers principales: player_web (para la carta FIFA) y jugador_web (para estadísticas reales). Cada router contiene endpoints que permiten realizar operaciones CRUD sobre sus respectivos modelos.

  • 📌 main.py → Entrada principal del sistema y enrutamiento general
  • ┣ 📁 router/player_web.py
    • 🔹 GET /players → Lista de jugadores FIFA
    • 🔹 POST /players → Crear nuevo jugador FIFA
    • 🔹 PUT /players/{id} → Modificar jugador FIFA
    • 🔹 DELETE /players/{id} → Eliminar jugador FIFA (soft delete)
  • ┣ 📁 router/jugador_web.py
    • 🔹 GET /jugadores → Lista de jugadores reales
    • 🔹 POST /jugadores → Crear nuevo jugador real
    • 🔹 PUT /jugadores/{id} → Modificar jugador real
    • 🔹 DELETE /jugadores/{id} → Eliminar jugador real (soft delete)
  • 📄 GET /planeacion, /diseno, /objetivo → Secciones documentales del sistema
  • 👤 GET /developer → Información del desarrollador

Wireframes / Mockups

Se diseñaron prototipos de interfaz que permiten anticipar la distribución visual, organización de botones, formularios y tarjetas de jugador. Estos diseños fueron creados con herramientas como Canva y se utilizaron como guía para implementar la interfaz final.
