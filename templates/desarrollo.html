🧪 Fase de Desarrollo

En esta etapa del proyecto integrador se implementó un sistema web completo que permite gestionar datos de jugadores de la UEFA Champions League 2021-2022, integrando un backend funcional con FastAPI y SQLModel, conectado a una base de datos PostgreSQL alojada en la nube (Supabase). A nivel frontend, se diseñó una interfaz HTML responsiva y visualmente amigable con estilos consistentes, navegación fluida y formularios interactivos para la creación, edición, consulta y eliminación lógica de jugadores y sus métricas.

Se construyó un sistema modular con rutas separadas para jugadores y cartas FIFA, validaciones asincrónicas, endpoints estructurados, y funcionalidades CRUD completas. El sistema fue desplegado de manera local y se documentó de forma clara para facilitar su evaluación y mantenimiento. Esta fase refleja la consolidación práctica de habilidades en bases de datos, backend en Python, integración con servicios externos (como Supabase), pruebas, control de versiones en GitHub y diseño de interfaces limpias y funcionales.

⚙️ Componentes exigidos en esta fase

  • Despliegue del sistema en un servidor online.
  • Conexión con una base de datos remota.
  • Repositorio en GitHub con commits organizados y descriptivos.
  • Pruebas de funcionamiento (testing).

💡 Requisitos técnicos específicos

  • Interfaz en HTML con navegación coherente entre páginas.
  • Estilo uniforme en tipografía, colores, botones y estructura.
  • Formulario para registrar objetos en la base de datos, incluyendo imágenes.
  • Funcionalidad para consultar por ID, nombre o característica.
  • Consulta general de todos los elementos registrados.
  • Edición total o parcial de registros existentes.
  • Eliminación lógica de registros (manteniendo historial).

  • Endpoint para mostrar la fase de diseño.
